2.REST api
Representational State Transfer의 약자로
REST는 HTTP에서 필요한 자원에 접근 할 때 웹의 장점을 활용하기 위한 아키텍쳐

	구성요소
HTTP Method
GET      - 데이터 조회
POST    - 데이터 추가
PUT       - 데이터 전체수정 
PATCH   - 데이터 일부 수정
DELETE - 정보삭제


URL - 데이터 접근

Representstion - 자원의 표현

REST API의 자원 표현을 위한 URL 설계규칙
소문자만 사용
하이픈(-)은 사용하지 않고 밑줄(_)을 사용
확장자를 표시하지 않음
